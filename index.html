Figma Blog_example를 100% 반영한 좁은 단일 칼럼 블로그 규격입니다. 디자인 토큰·컴포넌트·작성법을 한곳에 모았고, 모든 데모는 실제 style.css로 렌더링됩니다.

Guide

작성 규격 (Method)

완성본은 표준 HTML 문서가 아니라 호스트 페이지 안에 삽입되는 본문 조각입니다.

  • 출력은 <style>@import…</style> + <main class="page">로 시작해 </main>으로 끝낸다.
  • <!DOCTYPE>·<html>·<head>·<body>·<script> 금지. CSS만 @import로 로드.
  • 좁은 단일 칼럼(640px). 섹션 헤더는 .sec--bar 또는 .sec--quote.
  • 전체 규격은 저장소 CLAUDE.md 참고.
<style type="text/css">@import url("./style.css");</style>
<main class="page">
  <article> … </article>
</main>

Guide

SEO 검색최적화

네이버·구글이 선호하는 방식으로 블로그 조각을 작성하는 규칙입니다. 핵심 원칙: 조각(본문)은 <script> 없이 시맨틱 + Microdata로, 호스트 <head>엔 JSON-LD·메타로. (claude-seo 스킬 방법론 기반)

① 구조 & 시맨틱

  • 제목 위계 준수: 글 제목 h2.page-title → 소제목 h3.sec-title (건너뛰기 금지).
  • 본문은 <article>로 감싸고, 목록은 <ul>/<li>, 단락은 <p>로.
  • 장식용 글리프(큰 따옴표)는 aria-hidden="true".

② 구조화 데이터 (스크립트 없이)

조각엔 Microdata를 직접 넣습니다(구글·네이버 모두 인식). 본문 제품 광고는 Product+Offer, 글 전체는 BlogPosting.
<article itemscope itemtype="https://schema.org/BlogPosting">
  <h2 itemprop="headline">…</h2>
  <time datetime="2024-10-20" itemprop="datePublished">2024.10.20</time>
</article>

<a class="promo" itemscope itemtype="https://schema.org/Product">
  <div class="promo-title" itemprop="name">3단 근조화환 (기본형)</div>
  <div itemprop="offers" itemscope itemtype="https://schema.org/Offer">
    <span itemprop="price" content="59000">59,000원</span>
    <meta itemprop="priceCurrency" content="KRW">
    <link itemprop="availability" href="https://schema.org/InStock">
  </div>
</a>

③ 호스트 <head> — JSON-LD & 메타 (권장)

호스트(테마)에서 <head>를 편집할 수 있으면, 구글이 가장 선호하는 JSON-LD와 메타를 넣습니다. 제목 30~60자, 설명 120~160자, canonical·OG 필수.
<title>근조화환 보내기 전 체크리스트 | 도랑플라워</title>
<meta name="description" content="빈소 확인부터 당일배송, 리본 문구까지 — 근조화환 주문 전 꼭 확인할 핵심만 정리했습니다.">
<link rel="canonical" href="https://example.com/blog/geunjo-hwahwan">
<meta property="og:title" content="근조화환, 보내기 전에 이것만은 꼭 확인하세요">
<meta property="og:image" content="https://example.com/og.jpg">

<script type="application/ld+json">
{
  "@context": "https://schema.org",
  "@type": "BlogPosting",
  "headline": "근조화환, 보내기 전에 이것만은 꼭 확인하세요",
  "image": "https://example.com/hero.jpg",
  "datePublished": "2024-10-20",
  "dateModified": "2024-10-20",
  "author":    { "@type": "Organization", "name": "도랑플라워" },
  "publisher": { "@type": "Organization", "name": "도랑플라워",
                 "logo": { "@type": "ImageObject", "url": "https://example.com/logo.png" } },
  "mainEntityOfPage": "https://example.com/blog/geunjo-hwahwan"
}
</script>

④ 이미지

  • 모든 이미지에 서술적 alt(10~125자, 파일명·키워드 나열 금지).
  • 히어로(첫 화면=LCP)는 fetchpriority="high", 나머지는 loading="lazy" decoding="async".
  • 레이아웃 컨테이너에 aspect-ratio를 줘 CLS(밀림)를 방지(본 디자인 적용 완료).

⑤ 콘텐츠 품질 (E-E-A-T) & 네이버

  • 경험·전문성: 실제 경험·구체 사례·1차 정보(직접 촬영 이미지)로 작성. 일반론 나열 금지.
  • 작성자·날짜 노출(author, <time>)로 신뢰 신호.
  • 분량: 정보형 글은 충분한 깊이(권장 1,500자+), 검색 의도에 정확히 답하기.
  • 네이버: 명확한 제목·소제목, 본문 텍스트 충분, 이미지 alt, 모바일 가독성. 서치어드바이저에 사이트·사이트맵 등록은 호스트에서.
  • 내부 링크: 관련 글/상품으로 서술형 앵커 텍스트(“여기 클릭” 금지).
